Responsive Web Design

Responsive web design is an approach to web page creation that makes use of flexible layouts, flexible images and cascading style sheet media queries. The goal of responsive design is to build web pages that detect the visitor’s screen size and orientation and change the layout accordingly.

Advantages of Responsive Web Design

  • Optimal display of pages width all elements and full functionality.

  • A single URL for the regular version and page with adaptive layout, user redirection is not required.

  • Saving content, no duplication in different versions, possibility of SEO-optimization.

  • Improving behavioral metrics of the site.

Disadvantages of Responsive Web Design

  • High complexity of the process.

  • The Responsive website takes a longer time to download on the small screen.

  • A responsive website will operate as per the media queries that give information about the screen size it is being viewed on. Many older browsers do not support such media queries.

Grid layout

CSS Grid Layout is the most powerful layout system available in CSS. It is a two-dimensional system, meaning it can handle both columns and rows, unlike flexbox which is largely a one-dimensional system. You work with Grid Layout by applying CSS rules both to a parent element and to that elements children.

Advantages of Grid layout

  • A grid can add better alignment, balance, and equality in spacing for a much better visual experience.

  • The wireframe is directly implemented into the design phase.

  • It is a great way to keep track of nested grids (grids within the main grid).

  • There is enough organization to easily switch around pieces of content and features. With little to no design adjustments.

Disadvantages of Grid layout

  • Many older browsers don’t support grid technology, so this makes it difficult to view content in older browsers.

  • Potentially, grids can get in the way of creativity. While it can promote alignment and organization, some designs just call for more abstraction.

  • The use of a grid requires a decent amount of content and visual elements. Without them, there is really no reason for a grid.

Grid Gaps

Grid Area

This property will set the starting and ending positions for both the rows and columns of an item

Fraction

Bands can be defined using any length unit. Grid also introduces an additional length unit to help us create flexible grid tracks. The new - fr unit represents a fraction of the available space in the grid container.

Resporsive Web Design

  • Responsive Web Design is about using HTML and CSS to automatically resize, hide, shrink, or enlarge, a website, to make it look good on all devices (desktops, tablets, and phones).

  • Responsive Web Design is more comfortable for users because have optimal display of pages’ width all elements and full functionality.

Grid layout

  • grid-template-columns defines the number and sizes of the columns of the grid. Shorthand property is grid-column.
  • grid-template-rows defines the number and sizes of the rows of the grid. Shorthand property is grid-row.
  • grid-gap puts blank space between rows and/or columns of the grid.
  • grid-area is a shorthand for grid-row-start, grid-column-start, grid-row-end, and grid-column-end, all in one line.
  • repeat() define the number of rows and columns in a grid can take a function as a value.
  • minmax() define the sizing for automatically created rows or columns. We may want to give tracks a minimum size, but also ensure they expand to fit any content that is added.
